# 交互式问答引导

> new-subsystem Skill 的核心问答流程

## 前置条件

- 子系统基础模板已创建（init_subsystem.py）
- 已进入子系统工作区目录 `subsystems/<name>/`

## 问答流程

---

### Q1: 核心职责

**提示：**
> 现在我们需要定义子系统的核心职责。请列出本子系统需要完成的主要任务。

**示例问题：**
- 这个子系统的主要职责是什么？
- 它需要处理哪些核心业务？

**回答格式：**
```
| ID | 职责 | 描述 | 优先级 |
|----|------|------|--------|
| F-001 | 账务处理 | 记录和分类日常收支交易 | high |
| F-002 | 财务报告 | 生成月度/季度/年度财务报表 | high |
```

**写入位置：** `SPEC.md` - 核心职责章节

---

### Q2: 边界约束

**提示：**
> 明确子系统的边界——什么是属于本系统的，什么不是。

**示例问题：**
- 什么任务属于本子系统？
- 什么任务不应该由本子系统处理？
- 与其他子系统的分工是什么？

**回答格式：**
```
**属于本子系统范围：**
- 所有财务数据录入和处理
- 财务报表生成和分析
- 预算制定和执行监控

**不属于本子系统范围：**
- 合同条款法律审核（法务子系统）
- 产品定价策略（产品子系统）
- 投资决策（需人工确认）
```

**写入位置：** `SPEC.md` - 边界约束章节

---

### Q3: Agent 团队配置

**提示：**
> 本子系统需要哪些 Agent 角色？如何协作？

**示例问题：**
- 需要哪些 Agent 角色？
- 每个角色的职责是什么？
- 何时触发哪个角色？

**回答格式：**
```
| 角色 | 职责 | 技能 | 触发条件 |
|------|------|------|----------|
| 会计 | 日常账务处理、凭证制作 | 记账、分类、对账 | 交易记录、费用报销 |
| 出纳 | 资金收付、银行对账 | 支付、核对、现金管理 | 支付请求、银行流水 |
```

**写入位置：** `CLAUDE.md` - Agent 团队章节

---

### Q4: 决策权限

**提示：**
> 定义子系统的决策权限矩阵——哪些可以自主决定，哪些需要人工确认。

**示例问题：**
- 什么决策可以自主完成？
- 什么需要人工确认？
- 金额阈值是多少？

**回答格式：**
```
| 决策类型 | 权限级别 | 金额限制 | 说明 |
|----------|----------|----------|------|
| 日常记账 | 自主 | 无限制 | 自动处理 |
| 费用审核 | 自主 | <¥1,000 | 自动审核通过 |
| 费用审核 | 需确认 | ¥1,000-10,000 | 通知人工确认 |
| 费用审核 | 禁止 | >¥10,000 | 必须人工审批 |
```

**写入位置：** `CLAUDE.md` - 决策权限章节

---

### Q5: 服务定义

**提示：**
> 定义提供的服务和消费的服务。

**示例问题：**
- 本子系统向其他子系统提供什么服务？
- 需要消费其他子系统的哪些服务？
- 服务如何调用？

**提供服务的回答格式：**
```
| 服务ID | 服务名称 | 描述 | 调用方式 |
|--------|----------|------|----------|
| FIN-001 | 账户余额查询 | 查询指定账户余额 | inbox请求 |
| FIN-002 | 交易记录查询 | 查询交易历史 | inbox请求 |
```

**消费服务的回答格式：**
```
| 服务 | 提供方 | 用途 |
|------|--------|------|
| 合同查询 | 法务 | 审核付款依据 |
| 法律合规检查 | 法务 | 税务合规确认 |
```

**写入位置：**
- `CONTRACT.yaml` - provides/consumes
- `CAPABILITIES.yaml` - capabilities

---

```
> [用户输入: 账务处理、财务报告、预算管理...]

```

## 错误处理

- 用户跳过：记录为 "[待定]"，可后续补充
- 用户退出：保存当前进度，下次可继续
- 不确定：用 `[ ]` 标记，稍后确认
